# Data Flow Diagrams and User Stories

# Overview:


This section describes how data flows through the system and how users interact with the application. The Data Flow Diagram (DFD) illustrates the movement of data from the dataset to the database, then to Tableau for visualization, and finally to the user through the Flask web interface. User stories describe how different users such as doctors, policymakers, and individuals interact with the dashboard to analyze heart disease data and gain useful insights.


# Solution Requirements

Overview:

Solution requirements define the functional and non-functional needs of the system. The system should be able to store the dataset in a database, process data using SQL operations, create interactive visualizations using Tableau, and integrate dashboards into a web application using Flask. The system must also ensure efficient performance, easy usability, and reliable data visualization.

# Technology Stack

Overview:

The technology stack includes the tools and technologies used to build the project. The system uses SQL databases for storing data, Tableau for data visualization, Python and Flask for web integration, and HTML/CSS for the user interface. These technologies work together to analyze heart disease data and present insights through interactive dashboards.
